Executive hook

For pipeline operators and subsea engineering teams, rigid jumper integrity questions often appear before the project is ready for a full bespoke VIV or fatigue program. The practical need at that stage is simple: which jumper/current/gap combinations are likely benign, which are marginal, and which clearly deserve escalation?

The pack screens that early question using a consistent DNV-RP-F105 (2017) basis across a structured parametric matrix. In the current demo dataset, the rigid jumper subset covers 200 cases for an 8 in rigid jumper, spanning 5-40 m, currents from 0.2-1.0 m/s, and seabed or mid-water gap ratios from 0.5 through infinity.

The result is not a black-box verdict. It is a reviewable screen with natural frequency, reduced velocity, estimated response amplitude ratio, status category, and maximum allowable span output for each case.

What this gives a pipeline company early:
  • Fast triage of jumper geometries that remain in a likely acceptable screening band.
  • Clear identification of cases that enter cross-flow onset or lock-in regimes.
  • A repeatable basis for deciding where to spend detailed engineering effort.
  • Client-ready tables that can be reviewed internally without waiting for a long study cycle.

Methodology and code basis

The underlying demo follows a DNV-style freespan/VIV screening workflow, using the existing digitalmodel results pack labeled to DNV-RP-F105 (2017). For each case, the screen evaluates natural frequency, reduced velocity, onset bands, and an estimated response amplitude ratio for an early integrity triage.

  • Code basis: DNV-RP-F105 (2017) for free-spanning pipelines; the demo report also cites DNV-RP-C205 and DNV-OS-F101 as adjacent reference context.
  • Boundary condition in the sample data: pinned-pinned, with Cn = 3.56.
  • Dataset constants shown in source metadata: content density 800 kg/m³, steel Young's modulus 207 GPa, inline onset VR = 1.0, cross-flow onset VR = 3.0, lock-in window VR = 4.0-8.0.
  • Status interpretation used here: PASS for VR < 1.0; INLINE_ONLY for 1.0 ≤ VR < 3.0; FAIL_CF for cross-flow onset; FAIL_LOCKIN for lock-in conditions.

This is deliberately framed as a screening tool rather than a final fatigue assessment. Its value is consistency, speed, and traceable escalation logic.

Limitations and caveat

  • This brochure is based on the existing demo dataset and should be read as illustrative screening collateral.
  • It does not replace project-specific VIV fatigue analysis, detailed hydrodynamic modelling, support stiffness characterization, or design verification.
  • The source demo assumes simplified boundary conditions, current-only screening, and fixed threshold bands; real projects may require more detailed loading and response treatment.
  • Gap ratio infinity in the sample set represents a mid-water or unsupported screening condition in the demo matrix, not a universal physical installation state.
  • Installation feasibility, if required, should rely on owner-approved vessel particulars, certified motions/RAOs, and final marine engineering checks.
  • All outputs require qualified engineering review before use in procurement, design, operations, or integrity decisions.
