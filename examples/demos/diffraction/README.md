# Diffraction demos

## `prepare_vessel_decks.py` — diffraction decks from the vessel database

Builds a representative cross-scope set of **OrcaWave diffraction decks** from the
vessel database (`marine_ops.vessel_db`), bridging vessel particulars →
`hull_library` mesh → `hydrodynamics.diffraction` (#896, feeds #622).

```
cd digitalmodel
uv run python examples/demos/diffraction/prepare_vessel_decks.py
```

For each vessel it writes `decks/<vessel_id>/`:

| file | what |
|---|---|
| `hull.gdf` | parametric WAMIT mesh from particulars (gitignored — regenerated by the CLI) |
| `<Vessel>.yml` | the OrcaWave deck (mass/inertia from `vessel_db.mass_properties`, wave periods, headings, water depth) |
| `manifest.json` | provenance, mesh kind, mass basis, and **swap instructions** |

Representative set (one per hull form):

| form | example | mesh |
|---|---|---|
| box | SeaRose FPSO | parametric |
| ship | Yavuz (drillship) | parametric |
| full | Alaskan Frontier (VLCC) | parametric |
| placeholder | SSCV Sleipnir (semisub) | **mesh-to-supply** (columns/pontoons can't be a single hull surface) |
| ship | DCV Balder (monohull crane) | parametric |

### What's prepared vs. what you run

- **Prepared here (no license):** complete OrcaWave `.yml` decks + parametric meshes + mass/inertia.
- **Run there (licensed):** execute each `<Vessel>.yml` on a licensed OrcaWave/AQWA machine to solve for RAOs / added mass / damping / QTFs.

### Mesh fidelity & swapping

The parametric mesh is a **wall-sided, plan-form-tapered first-pass screening hull**
— good for relative screening, not class-approved hydrodynamics. To upgrade to
design-grade, drop a faired hull mesh in as `hull.gdf` (the deck references it by
that single filename — no other change needed). Semisub/spar decks ship with a
`PROVIDE_HULL_MESH.gdf` placeholder for exactly this reason.

Mass properties carry their provenance: radii of gyration are `cited` where public
else `estimated` with a named relation; VCG is flagged when unknown. See each
`manifest.json`.
