# Cylinder Re=100 — laminar vortex-shedding verification case

Reference OpenFOAM case backing the verification report
[`../../cylinder-re100-verification.html`](../../cylinder-re100-verification.html)
(digitalmodel issue #1166, epic #1161).

Known-answer verification of transient bluff-body CFD: the laminar Kármán vortex street behind a
circular cylinder, whose drag and shedding frequency have well-established benchmark values.

## Result

| Quantity | CFD | Benchmark (Williamson 1996) | Error |
|----------|-----|------------------------------|-------|
| Mean drag `Cd` | **1.344** | 1.33–1.37 | **−0.5%** |
| Strouhal `St = fD/U` | **0.1655** | 0.164 | **+0.9%** |
| Lift amplitude `Ĉl` | 0.325 | ~0.32–0.34 | in range |
| Mesh | 21,120 cells (O-grid) | — | non-orthogonality <0.001 |
| Solver | OpenFOAM v2312 `pimpleFoam`, laminar | — | — |

## Conditions

`U∞ = 1`, `ν = 0.01`, `D = 1` → `Re = U∞D/ν = 100` (laminar shedding regime, above the critical
`Re ≈ 47`).

## Two engineering points

1. **Body-fitted O-grid, not snappyHexMesh.** A snapped/stepped surface degrades the wall shear and
   hence the drag; the annular O-grid (generated by `gen_blockmesh.py`) is essentially orthogonal
   (non-orthogonality < 0.001), which is why `Cd` lands within 0.5%.
2. **Symmetry trigger.** A grid-aligned symmetric setup keeps the lift identically zero — the
   shedding instability cannot grow from a symmetric state. Because a circle is rotationally
   symmetric, the free stream is tilted **5°** (`0/U`), which is the same physical flow rotated:
   `Cd`/`St` are unchanged, but the grid symmetry is broken so the instability develops. The
   `forceCoeffs` drag/lift directions are rotated to match, so the coefficients are unbiased.

## Reproduce

Requires a solver-capable host (`uv run openfoam doctor`; source the OpenFOAM environment first).

```bash
cd <a writable copy of this case>
uv run python gen_blockmesh.py     # writes system/blockMeshDict (4-block annular O-grid)
blockMesh
pimpleFoam                          # transient to t=160; shedding saturates ~t=120 (~10–15 min)

# post-process: mean Cd + Strouhal (zero-crossing) + force-history figure
uv run python analyze_cyl.py . ./cyl_results
```

`analyze_cyl.py` reads the `forceCoeffs` history, takes the mean `Cd` over the saturated window
(t=120–160) and the Strouhal number from the lift up-crossings (a raw FFT bin is too coarse — see
the report), and writes `results.json` + `force_history.png`.
