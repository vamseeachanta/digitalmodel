1  Verification philosophy

The digitalmodel CFD capability is verified by known-answer cases. No domain workflow is considered production-ready until it reproduces a result with a known analytical or benchmark answer. Verification runs execute against real OpenFOAM v2312 — not mocked — so the numbers below come from converged solutions of the actual solver, not from stubs. Every report is an interactive capability page: charts support hover/zoom/series-toggling (Plotly), the transient cases carry scrubbable animations of the solved fields (the Kármán vortex street; the dam-break collapse), and each case states the realistic offshore use case it gates — hull friction, VIV screening, control-surface lift, green water & sloshing. Two flow regimes anchor the suite: a steady boundary-layer case (flat plate, Blasius similarity solution) and a transient bluff-body case (cylinder at Re = 100, vortex shedding). CFD without verification is decoration.

3  How the capability is built

The CFD stack is fully open-source and runs license-free on Linux — no Windows licensed host is required, unlike the OrcaFlex / AQWA / ANSYS workflows. The toolchain is Python-orchestrated end to end:

FreeCAD + gmsh → OpenFOAM ESI + olaFlow → PyVista / ParaView

The execution layer wraps this pipeline in fail-closed, config-driven components:

  • OpenFOAMRunner — fail-closed mesh → solve → VTK orchestration (PR #1163).
  • openfoam doctor — solver-readiness diagnostics for a host (PR #1181).
  • Engine routing on basename: openfoam / cfd (PR #1190).
Caught only by running real OpenFOAM. A latent runner bug was surfaced — and fixed — only because verification ran against the actual solver: the error-marker matched OpenFOAM's benign startup banner trapFpe: Floating point exception trapping enabled, false-positiving every successful run. Fixed in PR #1191. A mocked pipeline would never have exposed this.

Every case is config-driven, deterministic, and provenance-stamped; figures and metrics are computed directly from the converged solutions — no hand-entered values.

4  Engineering depth

Each case involved a modelling judgment that mattered more than raw mesh count.

Flat plate — leading-edge treatment dominates. A 4-iteration error-isolation study drove Cf agreement from ~30% to ~5% by identifying the leading-edge treatment (a 20 mm upstream symmetry run-in) as the controlling modelling decision — not mesh density and not domain size. The dominant error source was physical, not numerical.
Cylinder — body-fitted grid + seeded shedding. A body-fitted O-grid (not snappyHexMesh) keeps wall shear accurate, giving Cd within 0.5%. A grid-aligned symmetric setup suppresses shedding, so the freestream is tilted 5° to seed it — a circle is rotationally symmetric, so Cd and St are unchanged. Strouhal is extracted from lift zero-crossings because the FFT bin is too coarse.
Turbulent flat plate — resolved to the wall. Meshed to y+ ≈ 0.13 (viscous sublayer, no wall functions bridging the gap); the k-ω SST profile collapses onto the universal law of the wall (sublayer + log law + wake). The residual ~9% Cf offset vs the empirical correlations sits within their own 5–10% mutual scatter and the resolved near-leading-edge transition; the law-of-the-wall collapse is the stronger verification.
NACA0012 — lift generation. Angle-of-attack polar by freestream rotation (wind-axis forces); lift-curve slope 0.1037/deg, within 2.2% of experiment and ~5% below the ideal 2π/rad — the physically correct thickness/viscous deficit. The slope is incidence-independent, so the mesh's built-in geometric offset (recovered as a −1.4° zero-lift angle) doesn't affect the verification; the reference area uses the measured ~35-chord, not unit chord.
Dam break — the gate-release artifact is physics, not error. The CFD releases the column instantaneously; the experiment's diaphragm removal takes finite time, so the raw front leads the measurement at early T. The literature-standard +0.175 dimensionless time shift (used identically by the Lethe benchmark) brings every digitized Martin & Moyce point within 8.1%. Both raw and corrected metrics are gated in the regression test, and the uniform mesh is sized so the column edges fall exactly on cell faces — the initial water volume matches the analytic column to five digits and drifts by only 6×10−7 over the run.
Wave tank — the near-field is physics, not error. The gauge 0.35 wavelengths from the wavemaker reads +8% on wave height: generation carries evanescent modes that decay within about a wavelength, so wave-quality gates apply from x ≥ 8 m (standard NWT practice). In the established region the height holds within 4.7% with 1% boundary reflection — the absorption quality that makes long screening runs affordable.
Floating body — the period offset is the added mass. The measured heave period exceeds the waterplane-stiffness value by 18%; solving for the added mass gives Ca ≈ 0.39, inside the published range for rectangular sections. A solver that missed the radiation/added-mass physics would land on the hydrostatic period — the offset is gated as a physical band, not hidden as an error. Heave-only constraints turn the tutorial into a clean 1-DOF decay benchmark.
