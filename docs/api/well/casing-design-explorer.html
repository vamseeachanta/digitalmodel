Ratings, load profiles and design factors are produced by digitalmodel.well.tubulars.casing (API 5C3 / API 5CT product catalog) and digitalmodel.well.tubulars.casing_design (load cases, design factors, frac allowable, NACE sour screen, connections) and frozen by scripts/capabilities/build_casing_design_explorer.py; a CI test re-runs every product through the live checks so a published margin cannot silently drift. This page's JavaScript only reads the frozen JSON — no physics is re-implemented client-side. Design-factor minimums, the Barlow worked example, the NACE thresholds / grade windows and the connection-class facts follow Hansen (Devon Energy), EPA Hydraulic Fracturing Technical Workshop — Session 2: Well Design, 2011; rating formulas per API 5C3 / API 5CT; sour limits per NACE MR0175 / ISO 15156.
