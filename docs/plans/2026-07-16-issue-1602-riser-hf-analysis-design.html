1. Goal and boundaries

The program will establish one solver-neutral source of truth for drilling and completion/workover riser configurations. It will generate deterministic target-solver models for every eligible configuration and publish traceable normalized inputs and verified results to aceengineer/digitalmodel-riser-analysis. OrcaFlex will be the first licensed execution target; custom FEA and future solvers will use the same canonical cases through versioned adapters.

The first public release will include public-safe drilling inputs and explicitly synthetic completion reference cases. It will not publish raw OrcaFlex DAT/YAML files, private/client material, licensed standards content, embedded machine metadata, or results whose capacity basis lacks a valid citation.

Non-substitution rule: a synthetic completion reference case will not be presented as a recovered field, vendor, client, or measured configuration. Confidential completion models will remain withheld until a separately approved promotion decision establishes redistribution and disclosure safety.

4. Architecture and data flow

source discovery → source registry and classification → strict normalization into solver-neutral SI schema → engineering and release eligibility gates → analysis-case compiler → versioned target-solver adapter → deterministic target-specific projection → target-specific execution and attestation → typed result extraction and plausibility review → public-safe Parquet tables + dataset card → immutable HF revision and datasets-server verification

Component boundaries

  1. Registry: will inventory each candidate once, record its canonical/duplicate relationship, and fail closed when ownership, license, or provenance is unknown.
  2. Normalizer: will parse allowlisted fields, reject malformed row widths and non-finite values, convert declared units to SI, and retain source-field lineage.
  3. Case compiler: will combine one normalized configuration with explicit loading/environment data without filling unknown engineering values by guesswork.
  4. Adapter interface: will project the immutable generic case into a target model and emit target ID, adapter-contract hash, projection hash, and deterministic manifest. It will fail on unsupported semantics and will never back-write solver names or defaults into the SSOT.
  5. OrcaFlex v1 adapter and licensed runner: will execute statics and dynamics on an approved Windows host and will attest solver/DLL version and input/model hashes.
  6. Future solver adapter: custom FEA or another solver will consume the same case and analysis intent, map results to the same generic quantity/location/criterion contracts, and satisfy the same request/receipt/result evidence boundaries.
  7. Result extractor: will emit typed engineering outputs, failure records, and criterion identifiers; it will never replace failed runs with mock values.
  8. Publisher: will select release-approved rows, generate Parquet plus the dataset card, upload separately from digitalmodel-runs, and verify the immutable remote revision.

The target will be one public multi-config dataset: aceengineer/digitalmodel-riser-analysis. Each config will be a separate Parquet table with stable IDs and scalar viewer-compatible fields.

The dataset card will declare row counts, schemas, units, null semantics, license, source hashes, synthetic-case labeling, solver limitations, withheld fields/sources, and the exact code revision that generated each publication.

6. Eligibility and release gates

A configuration will be solver-eligible only when its schema, units, component relationships, boundary conditions, environment, and analysis requirements are complete. It will be publication-eligible only after the independent provenance, confidentiality, legal, engineering-plausibility, solver, and result-QA gates pass.

  1. Discover: record source without assuming eligibility.
  2. Classify: canonical, duplicate, public, synthetic, private, licensed, or derived.
  3. Normalize: strict parsing, unit conversion, lineage, and immutable hash.
  4. Validate: schema, physical bounds, relational integrity, citations, and leak scan.
  5. Compile: create an immutable analysis case and deterministic OrcaFlex projection.
  6. Solve: execute only with a real licensed solver; preserve failures.
  7. Review: apply non-tautological spot checks and domain plausibility limits.
  8. Publish: select only release-approved rows and verify remote rendering.

7. OrcaFlex strength and operability contract

Each eligible configuration will receive at least a baseline statics case and a configuration-driven operability case set. The detailed sweep axes and values will be frozen in the issue plan from traceable public/synthetic inputs rather than embedded as unexplained defaults.

Strength outputs

  • Effective tension, bend moment, curvature, and stress response at declared critical locations.
  • Upper/lower flex-joint response, tensioner force/stroke, and component-specific response where modeled.
  • Utilization only when the capacity equation, material basis, safety/design factor, units, revision, and Citation sidecar are complete.

Operability outputs

  • Vessel offset, wave heading/period/height, current profile, contents density, and top-tension coordinates used by the case.
  • Limiting sea state or offset, limiting criterion/location, flex-joint angle, tensioner stroke/margin, and operable/non-operable classification.
  • Operability fraction only when a publishable metocean scatter basis is available; otherwise the result will remain an envelope, not a fabricated probability.
Solver truth rule: dry-run, generated-model, unit-test, reduced-order, and mock outputs will carry distinct terminal types and will never be published as licensed OrcaFlex results.

8. Identity, determinism, and reproducibility

The chain will preserve source_sha256 → normalized_input_sha256 → generic_parameter_contract_sha256 → case_sha256 → solver_target_id/adapter/projection hashes → native model/result hashes → HF commit SHA. Stable IDs will be content-derived or deterministically assigned from canonical IDs. Every published result will join back to exactly one analysis case and normalized configuration.

The solver manifest will record the digitalmodel commit, adapter/schema version, OrcaFlex/DLL version, host role (not hostname/user), requested stages, timestamps, terminal status, warnings, and artifact hashes. Machine-specific absolute paths and credentials will never enter the manifest.

9. Privacy, licensing, and security

  • Allowlist-based field extraction will replace broad serialization of source models.
  • Legal and leak gates will reject usernames, hostnames, emails, absolute paths, project/client identifiers, secrets, restricted deny-list tokens, and unapproved URLs.
  • Standards citations will identify code/revision/section without reproducing licensed tables or prose.
  • Vendor/model names from the malformed drilling CSV will remain quarantined until original lineage and redistribution basis are resolved.
  • Completion reference cases will be publisher-owned synthetic configurations and will not be calibrated by copying values from withheld models.
  • HF authentication will remain environment/token-file backed. Tokens will never be printed, committed, or included in run evidence.

11. Verification strategy

Implementation will use TDD. Tests will be written and observed failing before production changes. Verification will include:

  • CSV row-width and strict-schema rejection tests, including the currently malformed 36-row drilling source.
  • Canonical-vs-duplicate detection and source-hash tests.
  • Unit conversion, physical-bound, relationship, and stable-ID/hash tests.
  • Confidentiality/leak fixtures covering source headers and embedded OrcaFlex metadata.
  • Normalized-case → generated-model mapping and deterministic regeneration tests.
  • Licensed-worker integration tests proving OrcFxAPI version, statics/dynamics execution, typed result extraction, and failure capture.
  • Independent engineering spot checks that do not reuse the implementation formula as the oracle.
  • Dataset-card/config/schema/row-count/hash validation and publisher dry-run tests.
  • Post-upload HfApi file/revision checks plus datasets-server validity and row queries for every config.
  • Mandatory legal scan and adversarial code/artifact cross-review before issue close.

12. Delivery slices and issue boundaries

  1. Data repair and source contract: repair or explicitly quarantine the malformed worldenergydata CSV; land the source registry and normalized-schema contract.
  2. Public-safe configuration set: assemble drilling configurations and independently authored synthetic completion reference cases.
  3. Case compiler and adapter interface: freeze generic cases and deterministic solver projections; implement OrcaFlex as the first target without coupling the SSOT to it.
  4. Licensed execution and extraction: run all eligible configurations and produce verified strength/operability tables.
  5. HF publication: build Parquet/card, upload, verify immutable revision and datasets-server rows, and trigger any registered rebuild only through approved secrets.

13. Design acceptance

  • The normalized solver-neutral model is the single source of truth.
  • Canonical parameters contain generic engineering state only; solver object names, files, hosts, licenses, and implicit defaults remain derived adapter concerns.
  • A custom FEA or future solver can be added through a reviewed adapter without changing canonical case identity or forking the SSOT schema.
  • Option 1 produces one public-safe HF projection and never uploads raw completion models.
  • Drilling and synthetic completion cases remain explicitly distinguishable.
  • Every published result is traceable through source, input, case, model, solver, result, and HF revision hashes.
  • Failed, mock, withheld, or unverified cases cannot masquerade as licensed solver results.
  • Strength utilization and standards-derived constants cannot publish without Citation sidecars.
  • All eligible configurations receive the agreed strength and operability analysis; ineligible sources remain visibly withheld with reasons.
