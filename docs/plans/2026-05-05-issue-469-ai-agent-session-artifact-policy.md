# Plan: digitalmodel #469 — Assess AI agent session artifact dirs across machines

**Issue:** https://github.com/vamseeachanta/digitalmodel/issues/469
**Status:** plan-review
**Tier:** T2 (cross-machine policy + gitignore)

## Context

Three agent ecosystems write working dirs into the digitalmodel repo root: `.codex/CuXXX/`, `.gemini/CuXXX/`, and `.claude/skills/{guidelines,meta,workflows}` plus `.claude/skills/.ecosystem-propagated`. The ACMA-ANSYS05 machine already gitignored `.codex/Cu*/` and `.gemini/Cu*/` (commit `e4b1ea6f`); other machines need parity. The `.claude/skills/` pointer files are still **OPEN** — the issue asks whether to commit them (so other machines get correct skill pointers on clone) or gitignore (regenerated by propagation).

Critical context: the per-session `CuXXX` raw logs are already aggregated to `workspace-hub/.claude/skills/session-logs/aggregated/` nightly — no value re-committing them. The pointer files (`.claude/skills/{guidelines,meta,workflows}`) are ASCII text containing relative symlink targets; they're set by ecosystem propagation. If gitignored, a fresh clone has no pointers until propagation runs, breaking immediate-skill-discovery on clone.

The decision must be **uniform across all machines** to avoid drift and stale committed pointer files conflicting with on-disk regeneration.

## Plan

1. **Audit current state on every machine the digitalmodel repo is cloned.** New script `scripts/agent-artifact-audit.sh` (kept in workspace-hub since it spans multiple repos, but linked from this issue):
   - List `.codex/Cu*/` count and total size
   - List `.gemini/Cu*/` count and total size
   - Stat `.claude/skills/{guidelines,meta,workflows}` — symlink? regular file? pointer text?
   - Stat `.claude/skills/.ecosystem-propagated` and emit its date
   - Confirm `.gitignore` patterns
   The script writes machine-id-stamped output that gets pasted into this issue per-machine. No code change yet.

2. **Lock the policy decision.** Recommendation (justified below): **gitignore everything**, regenerate pointers via propagation on clone.
   - **Why gitignore the `.codex/Cu*/` and `.gemini/Cu*/` dirs**: aggregated nightly, transient, 50+ accumulate. Already done on ACMA-ANSYS05.
   - **Why gitignore the pointer files**: they encode machine-specific paths (`../../../.claude/skills/_internal/`). Committing creates a maintenance treadmill where the canonical-on-disk and committed values diverge across machines.
   - **Compensating control**: add an idempotent `scripts/setup-claude-skills.sh` that regenerates the pointer files on clone, invoked from a one-time install step or a `make setup` target. This makes the gitignore safe by ensuring fresh clones reach the right state without manual intervention.

3. **Author the regeneration script.** New file `scripts/setup-claude-skills.sh`:
   - Computes relative path from `digitalmodel` repo to `workspace-hub/.claude/skills/_internal/`
   - Writes `.claude/skills/{guidelines,meta,workflows}` pointer files with the correct relative target
   - Writes `.claude/skills/.ecosystem-propagated` with today's date in YYYYMMDD
   - Idempotent: re-running overwrites, doesn't error
   - Exit non-zero with a clear message if `workspace-hub/.claude/skills/_internal/` not found

4. **Update `.gitignore`.** Edit (or create) `.gitignore` at digitalmodel repo root, add:
   ```gitignore
   # AI agent session artifacts (regenerated by tooling, not source-controlled)
   .codex/Cu*/
   .gemini/Cu*/
   .claude/skills/guidelines
   .claude/skills/meta
   .claude/skills/workflows
   .claude/skills/.ecosystem-propagated
   ```
   Confirm patterns don't shadow legitimately-tracked `.codex/` config or `.claude/skills/_internal/` content.

5. **Document in CONTRIBUTING.** Append a short section to `CONTRIBUTING.md` (or `README.md` if no CONTRIBUTING) explaining: "On fresh clone, run `bash scripts/setup-claude-skills.sh` to regenerate AI-agent skill pointer files."

6. **Tests.**
   - `tests/scripts/test_setup_claude_skills.sh` (shell test): run the script in a temp dir, assert the four files land with the right content
   - Manual cross-machine verification: run audit script on ACMA-ANSYS05, ACMA-ANSYS04, ace-linux-1, ace-linux-2; paste outputs into the issue
   - Confirm `git status` clean after running `bash scripts/setup-claude-skills.sh` (i.e., the regenerated files are gitignored)

7. **Smoke check.** `bash scripts/setup-claude-skills.sh && git status` — must show no new tracked files. `cat .claude/skills/guidelines` — must be readable pointer text.

## Acceptance Criteria

- [ ] Audit run on ≥ 4 machines, results posted to issue thread
- [ ] `.gitignore` updated with the six new patterns
- [ ] `scripts/setup-claude-skills.sh` lands, idempotent, regenerates pointer files
- [ ] CONTRIBUTING / README documents the post-clone setup step
- [ ] `git status` clean after running the setup script on a fresh clone
- [ ] No previously-tracked artifact dirs remain tracked (run `git rm -r --cached .codex/Cu* .gemini/Cu* .claude/skills/{guidelines,meta,workflows} .claude/skills/.ecosystem-propagated 2>/dev/null` if any are tracked)
