## Findings

1. Removal is justified and correctly bounded: both packages are unconditional
   root dependencies/direct lock edges, no tracked Python consumer exists, and
   the dashboard's separate hiredis pin remains out of scope.
2. **MAJOR:** CI and clean-host `uv pip install --no-sources -e ".[dev]"`
   resolve current metadata rather than consume `uv.lock`; `uv lock --check`
   proves freshness but not graph identity.
3. **MAJOR:** `uv.lock` is not cross-machine portable because assetutilities is
   an editable `../assetutilities` source. The plan alternates between that
   local graph and an unlocked registry graph, while CI does not pin uv 0.11.25.
4. **MAJOR:** baseline worktrees intentionally lack the assetutilities sibling
   yet request locked sync; expected new contract-test collection also conflicts
   with the rule that any collection difference blocks. Fresh environments and
   caches must be explicit.
5. **MINOR:** durable RED evidence should bind pre-change commit and test blob,
   record collection/expected failing nodeids, and precede separate GREEN
   checkpoints for dependency/lock and workflow slices.
6. Compiler-free scope is mostly honest, but the proof should bind exact OS,
   architecture, Python patch versions, uv version, and the graph used by CI.
7. **MAJOR:** cross-machine handoff must commit and push all resumable changes
   plus the handoff before claim release, and pickup must verify exact remote
   head.
8. **MAJOR:** ASCP claim acquisition needs delayed re-read verification,
   renewal verification, and a final ownership check before push to mitigate
   check-then-post races.
9. **MAJOR:** approval must bind the exact reviewed plan blob, approval source,
   and review evidence; executor preflight must verify the live label and SHA.

## Blockers

- Choose one reproducible graph for CI, clean-host proof, and paired regression
  runs; consume it and pin uv 0.11.25.
- Resolve the assetutilities sibling-source contract explicitly.
- Correct baseline isolation and enumerate allowed new test-node collection.
- Require commit/push-before-release, exact remote pickup, and race-aware claim
  verification.
- Bind `.planning/plan-approved/701.md` and implementation preflight to the
  approved plan/review blob SHAs and live GitHub approval label.
