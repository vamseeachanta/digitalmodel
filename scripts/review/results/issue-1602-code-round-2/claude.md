### Verdict: MAJOR

### Summary
Adversarial review of digitalmodel #1602 Task 1 (parent-contract validator/composer, full branch diff; 45/45 tests pass in-repo). Input/output hardening is genuinely strong: dup-key-rejecting SafeLoader, O_NOFOLLOW + dev/ino TOCTOU cross-checks, size caps, lexical+resolved+component-walk symlink containment, output-alias rejection, atomic fsync'd emit with temp cleanup — all backed by real adversarial tests. Exact-semantics pinning (frozen hashes for derivations, release membership, host envelope extensions, ref-count census, crosswalk, guard predicates) defeats most mutations. However, two empirically confirmed bypasses violate the declared merge-only/no-overwrite/exact-membership contract on unpinned surfaces: (1) _merge_owners (src/digitalmodel/contract_validation/riser_parent_contract.py:222-227) implements owner 'amend' as dict overwrite, so a mutated host component (hashes refreshed, same threat model as the suite's own mutation tests) can replace a base owner's issue URL with an attacker URL and wipe its outputs, and the result validates and is emitted — contradicting the declared directive merge_by_owner_key_and_append_strengthening_amendment (riser_contract_rules.py:39); (2) riser_contract_rules.py:109-111 lets a singular 'consumer' key shadow the 'consumers' list, so consumers:["totally_bogus_owner"] on a new host envelope passes membership validation and ships in the composed output. Additionally, several invalid inputs crash with raw KeyError/ValueError during composition (riser_parent_contract.py:323,325) before the corresponding clean validation checks run, making those checks unreachable dead code for those inputs — fail-closed is preserved (no output emitted) but the ContractValidationError contract is broken. Test adequacy is high for paths/atomicity/exact-semantics but missed exactly the unpinned owner-content and new-envelope-content surfaces where the bypasses live. VERDICT: MAJOR

### Issues Found
- MAJOR — merge-only violated for owners: src/digitalmodel/contract_validation/riser_parent_contract.py:222-227 (_merge_owners) does {**base_owner, **host_owner} for keys in effective_owners.amend. Reproduced counterexample: set host owners.drilling_workflow.issue='https://github.com/attacker/hijack/issues/1' and outputs=[] in issue-1602-riser-host-motion-contract-v3.yaml, refresh manifest sha256s -> validate_parent_contract succeeds and emits composed owners.drilling_workflow with the attacker URL and empty outputs. No frozen-hash or content check pins base-owner fields (CHILD_CROSSWALK/_validate_children check owner NAMES only), contradicting declared directive merge_by_owner_key_and_append_strengthening_amendment (riser_contract_rules.py:39).
- MAJOR — membership check bypass via key shadowing: src/digitalmodel/contract_validation/riser_contract_rules.py:109-111 (_validate_envelopes) replaces the 'consumers' list with ['consumer'] when the singular key is present. Reproduced counterexample: add consumers=['totally_bogus_owner'] plus consumer=<producer> to host interface_envelopes.floating_host_identity, refresh hashes -> validation passes and the bogus unresolved consumer is emitted in the composed contract. Host interface_envelopes content (unlike existing_envelope_extensions) has no exactness hash, so this surface is attacker-mutable.
- MEDIUM — unreachable validation / raw-exception fail-closed: unknown child_acceptance_extensions issue raises KeyError at riser_parent_contract.py:325 (_compose_assurance) because composition runs before _validate_children (riser_contract_rules.py:295-303), whose 'child obligation has no owner crosswalk' error is unreachable for this input; likewise obligations['public_release'].remove('exact_13_config_schemas') at riser_parent_contract.py:323 raises raw ValueError before _validate_composition's 'declared removal does not exist in base' check. CLI (scripts/validation/validate_riser_parent_contract.py:31-34) catches only ContractValidationError, so these produce tracebacks. No output is emitted, so fail-closed holds, but the typed-error contract is broken and the affected checks are dead code for these inputs.
- MINOR — missing top-level keys throughout composition (e.g. root['contract_version'], base['planning_validation'], envelope['minimum_bindings']) raise raw KeyError rather than ContractValidationError; same crash-not-clean-error class as above.
- MINOR — riser_parent_contract.py:348 (_atomic_emit): output.parent.mkdir(parents=True, exist_ok=True) sits outside the try block, so an OSError there escapes untyped instead of 'cannot atomically emit output'.
- MINOR — riser_manifest_rules.py:47-57 (resolve_bound_path): os.path.commonpath raises uncaught ValueError on Windows cross-drive paths, and the containment root is manifest.parent.parent — if the manifest is ever placed at a shallow directory the allowed tree widens accordingly.
- MINOR — riser_parent_contract.py:151-153 (_read_regular): all OSErrors (including EACCES) are reported as 'input must be regular non-symlink', a misleading diagnostic.

### Suggestions
- Enforce strengthening-only owner amendment: in _merge_owners, for keys in the amend set, require equality on pre-existing fields (or restrict host owner entries to an allow-listed {'issue': must-equal-base, 'amendment': new-key-only} shape), and/or pin host['owners'] with a frozen record hash exactly as done for existing_envelope_extensions (HOST_EXTENSION_HASH pattern).
- In _validate_envelopes, reject envelopes that declare both 'consumer' and 'consumers' (or validate the union of both), and add a frozen-hash pin or exact-key-set check for host['interface_envelopes'] content so new-envelope surfaces get the same mutation resistance as extensions.
- Convert composition-time KeyError/ValueError sites into ContractValidationError (guarded lookups in _compose/_compose_assurance), or run the structural validations that guard them before composing, so _validate_children and the explicit-removal existence check become reachable again; add regression tests asserting typed errors for unknown crosswalk issues and missing removal targets.
- Add mutation tests for the two confirmed counterexamples (owner issue/outputs overwrite; consumer/consumers shadowing) mirroring the existing test_riser_parent_contract_semantics.py style.
- Move output.parent.mkdir inside _atomic_emit's try block and wrap commonpath calls to convert ValueError into ContractValidationError for portability.

The suite's own mutation tests and the frozen-hash constants imply yes — in which case unpinned surfaces (owners content, new host envelope content, aux bound-file paths) need the same treatment; if no, what binds the manifest itself to the approved artifacts?

- Is _validate_children's 'child obligation has no owner crosswalk' branch intended to be reachable? Currently _compose_assurance KeyErrors first for the same input.
- Which base owner fields, if any, is a host amendment permitted to change? The declared directive says append-strengthening, but the implementation permits arbitrary replacement — which is authoritative?
